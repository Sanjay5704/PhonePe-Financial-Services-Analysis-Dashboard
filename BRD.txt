BUSINESS REQUIREMENTS DOCUMENT (BRD)

PHONEPE FINANCIAL SERVICES ANALYSIS DASHBOARD

USING DIGITAL TRANSACTION DATA

---

CLIENT

PhonePe is one of India’s leading digital payment and financial services platforms, enabling millions of users to perform secure and convenient financial transactions.

The platform offers a wide range of services such as:

- Money Transfers
- Mobile Recharges
- Utility Bill Payments
- Insurance Services
- Loan-related Financial Services

With increasing adoption of digital payments and financial products, there is a growing need to analyze transaction trends, user engagement, and service-level performance to support data-driven business decisions.

---

DATA SOURCE

Dataset Used:
PhonePe Transaction Dataset

Description:
The dataset contains structured digital transaction data related to different financial services available on the platform.

The dataset is used to analyze service-level transaction behavior, identify usage patterns, and visualize key metrics across multiple categories.

Key Data Areas Include:
- Transaction Categories
- Transaction Counts
- Transaction Values
- Service Types
- User Activity
- Time-Based Transaction Trends
- Payment / Transaction Failure Details

Typical Fields Include:
- Transaction ID
- Transaction Type
- Service Category
- Transaction Count
- Transaction Amount / Value
- Transaction Date / Time Period
- User Count / Engagement Metrics
- Payment Status / Failure Reason

---

OBJECTIVE

The objective of this Business Requirements Document is to define the analytical and dashboarding requirements for building a Power BI-based Financial Services Analysis Dashboard using PhonePe transaction data.

The dashboard is intended to help stakeholders:

- Monitor digital transaction growth and service adoption
- Analyze transaction behavior across multiple service categories
- Track user activity and service usage patterns
- Identify high-performing and low-performing financial services
- Understand recurring digital payment behavior
- Present actionable insights through a structured analytics dashboard

---

## KEY PERFORMANCE INDICATORS (KPIs)

### 1. Total Transactions
Measures the total number of transactions recorded across the platform.

**Formula:**  
Total Transactions = SUM(Transaction Count)

---

### 2. Total Transaction Value
Measures the total monetary value processed through the platform.

**Formula:**  
Total Transaction Value = SUM(Transaction Value)

---

### 3. Total Users / User Activity
Measures the number of active users engaging with the platform over a given period.

**Formula:**  
Total Users = SUM(User Count)

---

### 4. Successful Transactions
Measures the total number of successfully completed transactions.

**Formula:**  
Successful Transactions = COUNT(Transaction ID where Status = "Success")

---

### 5. Failed Transactions
Measures the number of unsuccessful or failed transactions.

**Formula:**  
Failed Transactions = COUNT(Transaction ID where Status = "Failed")

---

### 6. Service-Wise Transaction Distribution
Measures the contribution of each financial service category such as:

- Money Transfer
- Recharge & Bills
- Insurance
- Loan

This KPI helps identify the most actively used services.

---

### 7. Transaction Trend Over Time
Tracks how transaction activity changes across time periods.

Used to identify:
- Growth trends
- Seasonal patterns
- Service adoption behavior

---

### 8. User Trend Over Time
Tracks platform engagement and how user participation changes over time.

This KPI helps understand:
- User growth
- Repeat usage
- Platform activity levels

---

### 9. Failure Reason Distribution
Measures and categorizes reasons for failed or incomplete transactions.

This helps in:
- Identifying friction points
- Improving transaction reliability
- Understanding service issues

## FUNCTIONAL REQUIREMENTS

The dashboard should meet the following functional requirements:

### Dashboard & Reporting Requirements
- Provide a multi-page interactive dashboard
- Include separate views for:
  - Home Overview
  - Insurance
  - Loan
  - Money Transfer
  - Recharge & Bills
- Display service-level transaction insights clearly

### Filtering & Interaction
- Ability to filter data by:
  - Time / Date
  - Service Category
  - Transaction Type
  - Payment Status (if applicable)

### Analytical Features
- KPI cards for quick performance summary
- Line charts for trend analysis
- Pie / donut / bar charts for service distribution
- Category-level and comparative visualizations
- Failure trend visualization

### Navigation
- Clear and user-friendly navigation between dashboard pages
- Consistent layout and design across all sections

## NON-FUNCTIONAL REQUIREMENTS

The dashboard should also satisfy the following non-functional requirements:

- Dashboard should be easy to use and visually clear
- Visuals should support quick business understanding
- Dashboard performance should remain acceptable during interaction
- Data displayed should be accurate, consistent, and reliable
- Layout should be suitable for business presentation and stakeholder review
- Design should remain clean, readable, and professional

## DATA TRANSFORMATION REQUIREMENTS

The following data preparation steps are required before dashboard development:

- Remove duplicate or inconsistent records
- Handle missing / null values appropriately
- Ensure correct formatting for:
  - Dates
  - Transaction counts
  - Numeric values
- Standardize category and service labels
- Create calculated fields / measures for KPIs
- Build proper relationships between tables (if using multiple tables)
- Create time-based fields for trend analysis
- Aggregate service-level and category-level metrics for reporting
